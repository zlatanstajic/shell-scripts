# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A collection of standalone Bash utility scripts for a Unix dev workflow. No build step, no test framework, no package manager. Each script in `src/` is run directly with `bash <script>.sh`.

## Commands

```bash

bash src/scripts/dev-setup.sh -nu 1 -na "Example issue name"
bash src/scripts/php-switch.sh -v 8.1       # or no args for interactive numeric pick
```

Single source of truth for flags/usage: the per-script flags/usage reference (each script rendered as a collapsible `<details>` block: title + source path in the `<summary>`, sanitized `-h` body inside) is GENERATED by `gen-docs.sh` into one marked region in `README.md` (`<!-- BEGIN/END GENERATED: command-reference -->`) and into `docs/_includes/command-reference.md` (consumed by `docs/scripts/reference.md`). The README per-script collapsible entries are entirely generated â€” there is no hand-maintained `<details>` prose in the README anymore. Everything else is hand-maintained and NOT generated: this `## Commands` block and the `docs/scripts/*.md` Parameters tables (separate Jekyll pages). To change a script's flags/usage, edit that script's `Help()`/`GetArguments` and run `bash src/scripts/gen-docs.sh` â€” never hand-edit the generated region. `bash src/scripts/gen-docs.sh --check` reports drift (advisory in CI and the pre-commit hook).

Lint: run `shellcheck src/scripts/*.sh src/lib/common.sh` if available (advisory only â€” CI runs it with `continue-on-error: true` and does not block on warnings). EditorConfig enforces 2-space indent, LF, final newline, 80-col max for `*.sh`.

Test: run `bash tests/run.sh` (whole suite) or `bash tests/run.sh tests/test_common.sh` (one file). See **Testing** below.

CI/hooks: [`.github/workflows/ci.yml`](.github/workflows/ci.yml) runs on push to `master`/`rework` and on every PR â€” two jobs: an advisory `shellcheck` (non-blocking) and the test suite (hard gate). The same checks run locally via the versioned [`.githooks/pre-commit`](.githooks/pre-commit) hook (tests gate, shellcheck advisory); enable once per clone with `git config core.hooksPath .githooks`, bypass with `git commit --no-verify`.

Docs site: [`docs/`](docs/) is a Jekyll/GitHub Pages site published at [zlatanstajic.github.io/shell-scripts](https://zlatanstajic.github.io/shell-scripts/); keep it in sync when script behavior changes.

## Architecture

**Shared library.** All logic common to scripts lives in [`src/lib/common.sh`](src/lib/common.sh). Each script lives in `src/scripts/` and sources the library via `source "$SCRIPT_DIR/../lib/common.sh"` after computing `SCRIPT_DIR`. `common.sh` is not executable on its own and depends on `SCRIPT_NAME` being set by the caller. Reuse its helpers instead of duplicating: `End` (terminate OK/error), `RunScript` (Ctrl-C trap + run main), `EchoBold`, `LogInfo`/`LogWarn`/`LogError`, `UserInput`, `DoYouWishToProceed`, `RunOrEcho` (dry-run-aware command wrapper: when `DRY_RUN=1` prints a `printf %q`-quoted `[dry-run] would: â€¦` line to STDERR and returns 0 without executing, else runs `"$@"`), `ConfirmOrAbort` (interactive gate: returns 0 under `DRY_RUN=1`/`ASSUME_YES=1`, else prompts and `End 0`s on decline), `MissingRequiredArguments`, `UrlEncode`. The `DRY_RUN`/`ASSUME_YES` globals default to `0` in `common.sh` for `set -u` safety; each script re-assigns them after its `.env` source (flag-driven only, never `.env` keys).

**Script skeleton** (every script in `src/scripts/` follows this exact shape â€” match it for new scripts):
1. Shebang + header comment block (Script name / Description / Parameters / Author / License).
2. `Variables` section: set `SCRIPT_NAME` and `SCRIPT_DIR` via `basename`/`dirname` of `readlink -f "$0"`, then `source "$SCRIPT_DIR/../lib/common.sh"`, then `set -u`.
3. `Help` function (uses `EchoBold`, lists `-x/--xxxx` flags).
4. `GetArguments` function: `while`/`case` loop over `$@`, `-h|--help` short-circuits to `Help; End 0`, validates required args, ends via `MissingRequiredArguments` when missing.
5. Task-specific functions.
6. `Main` function orchestrating the flow.
7. `Execution` section at file end: startup echo, then `RunScript Main "$@"`.

**Configuration via `.env`.** `dev-setup.sh`, `backup.sh`, `decrypt-env-files.sh`, `git-copy.sh`, `hash-filenames.sh`, and `restore-vscode-folder.sh` each source an optional `.env` from the project root â€” `$PROJECT_ROOT/.env`, where `PROJECT_ROOT` resolves to the repo root (`dirname` of `dirname` of `SCRIPT_DIR`, i.e. two levels up from `src/scripts/`) (copy [`.env.example`](.env.example) from the repo root to `.env`) over in-script defaults. The `.env` is *sourced as shell*, not parsed â€” any shell in it executes. Never commit a real `.env` (it is gitignored). `.env.example` documents the keys per script: `dev-setup.sh` (`BRANCH_PREFIX`, `REQUEST_PREFIX`, `ISSUE_BASE_PATH`, `GITLAB_ASSIGNEE_ID`), `backup.sh` (`BACKUP_LOCATION` + system/vscode/projects/deployments source paths, `ENV_FILES_PASSWORD`), `hash-filenames.sh` (`HASH_FILENAMES_FILE_EXTENSIONS`), `git-copy.sh` (`GIT_COPY_TARGET_DIRECTORY_PATH`), `restore-vscode-folder.sh` (reuses `backup.sh`'s `BACKUP_LOCATION` and `PROJECTS_DESTINATION_FOLDER_NAME`), `decrypt-env-files.sh` (reuses `backup.sh`'s `BACKUP_LOCATION`, `PROJECTS_DESTINATION_FOLDER_NAME` and `ENV_FILES_PASSWORD`), `splice-images.sh` (`SPLICE_IMAGES_FILE_EXTENSIONS`), `splice-videos.sh` (`SPLICE_VIDEOS_FILE_EXTENSIONS`), `tampermonkey-install.sh` (`TAMPERMONKEY_REPO_BASE_URLS`).

**Resilience pattern.** Scripts degrade gracefully when *optional* tools are absent: clipboard (`xclip`), opening URLs (`xdg-open`), PHP Apache module / `systemctl`, directory-tree copies (`rsync` â†’ `cp -r` fallback), privileged create/remove (`sudo` fallback), `git-copy`'s `zip`, `backup`'s `openssl` AES-256-CBC `.env` encryption, `hash-filenames`' `jq` mapping, and `tampermonkey-install`'s `curl` URL probe + GitHub token lookup (`$GH_TOKEN`/`$GITHUB_TOKEN`/`gh auth token`, used only to authenticate probes of private github.com repos) all `command -v`-guard and `LogWarn` instead of failing. By contrast, `splice-images.sh`/`splice-videos.sh` treat `ffmpeg` and `ffprobe` as *hard* dependencies â€” they `End 1` when either is missing, and `decrypt-env-files.sh` likewise treats `openssl` as a *hard* dependency (`End 1` when absent) since decryption is its entire purpose, unlike `backup`'s soft warn+skip.

**Testing.** A zero-dependency, pure-bash harness lives in `tests/` (no `bats`/`shunit2`). [`tests/run.sh`](tests/run.sh) sources [`tests/lib/assert.sh`](tests/lib/assert.sh) then every `test_*.sh` found recursively under `tests/` (sorted), prints a âś“/âś— line per assertion, and exits non-zero on any failure (CI-gate ready). Layout: shared-library tests at the `tests/` root, per-script tests under `tests/scripts/`. `assert.sh` exposes `assert_eq`, `assert_contains`, `assert_match`, `assert_exit` (captures command output into `$ASSERT_OUTPUT` via a temp file), the shared `TESTS_RUN`/`TESTS_FAILED` counters, and a resolved `REPO_ROOT`. Existing files: [`tests/test_common.sh`](tests/test_common.sh) unit-tests `common.sh` helpers (`exit`-calling ones like `End`/`MissingRequiredArguments` run in subshells); [`tests/scripts/test_generate_password.sh`](tests/scripts/test_generate_password.sh) drives `generate-password.sh` as a subprocess. To add a test, drop a `tests/scripts/test_<name>.sh`, use `$REPO_ROOT` for paths, route `exit`-calling code through `assert_exit` in a subshell, and assert whole-script behavior via `bash "$SCRIPT"` + exit code + `$ASSERT_OUTPUT`. Capture script output through a temp file, not a pipe: `generate-password.sh` can leave a `tr < /dev/urandom` reader holding a pipe open, hanging `| sed` / `$()` readers on EOF (a latent bug in the script â€” `pw=$(bash generate-password.sh)` is also affected). Coverage so far is `common.sh` + `generate-password.sh`; the other scripts (git/ffmpeg/filesystem side effects) need mocks/fixtures and are untested.

## Deployment model

Users mirror the whole `src/` dir to `deploy/versions/<version>/` (`deploy/` is gitignored) and edit/run from there, so they can `git pull` upstream without clobbering local edits. Because scripts in `scripts/` `source` `../lib/common.sh` by relative path, **`lib/` must travel alongside `scripts/`** â€” copy all of `src/`, never a single script.

Alternatively, [`install.sh`](install.sh) symlinks the user-facing scripts onto `PATH` (under their bare names) for a one-command install that coexists with the `deploy/` workflow. The installed commands are plain symlinks: each script resolves itself via `readlink -f "$0"`, so `SCRIPT_DIR` lands back in the clone's `src/scripts/` and `../lib/common.sh` + `PROJECT_ROOT/.env` still resolve â€” which means **the clone must stay put** (move it and re-run `install.sh` to re-point the links) and `src/lib/` must remain beside `src/scripts/`. The scripts are tracked executable (git mode `100755`) so a fresh clone yields runnable bare-name commands; the installer also defensively `chmod +x`es each target. [`uninstall.sh`](uninstall.sh) removes only the symlinks resolving into this repo's `src/scripts/`. `gen-docs.sh` is a maintainer tool and is excluded from both.

## Conventions

- Functions are PascalCase; the entry point is `Main`, wrapped by `RunScript`.
- Edit scripts in `src/` only, never in a `deploy/` copy.
- Branch off `master` as `issues/<num>_short_description` (see [CONTRIBUTING.md](CONTRIBUTING.md)).
- Test a change by running the affected script with `-h` and through a normal run on a Unix-like system.
